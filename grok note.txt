Where It’s Fragile (the parts that will bite you first)

Model size is the #1 bottleneckllama3.2 (1B/3B) + qwen2.5:3b are simply too small for reliable strict JSON + reasoning. They hallucinate structure constantly. Your massive deterministic fallback is saving you right now, but you’re fighting the model instead of leveraging it.
Parse JSON node is a 400-line regex monster
It works today, but it’s unmaintainable. One small change in user phrasing and half the regexes break. You basically re-implemented a mini parser because the LLM can’t be trusted.
No real error recovery loop
If the heavy planner also fails or confidence is low, it just errors. Real agents retry or ask clarifying questions.
Hard-coded knowledge & lists
Slovakia capital, top-8 capitals, etc. — cute, but will age badly and break when user asks for “10 largest cities in Europe”.
Write verification only checks space-after-letters
Everything else passes even if content is nonsense. The gate is too narrow.
Minor n8n gotchas
Using require('fs') inside a Code node is risky (n8n can disable it in newer versions).
The permission webhook assumes a separate service at port 4000 — easy to forget.
No folder creation tool (only files).
