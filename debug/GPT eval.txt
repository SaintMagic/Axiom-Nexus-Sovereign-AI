Model size handling

If you’re pulling something like:

download time and disk usage matter.

A 7B model can easily be 4–8 GB, which will surprise users if not shown clearly.

Ollama service control

Ideally the stack should ensure:

ollama serve

is running before any request hits the model.

Otherwise the first inference will fail silently.

n8n startup race

Your bridge depends on n8n being fully ready. The installer/start script should probably include a readiness check such as:

wait for localhost:5678

before allowing the agent to send requests.

Right now the AI → n8n interaction seems to be single-endpoint delegation:

Long-term it will scale better if the model produces structured tool calls like:

  "arguments": {...}
}

Then n8n acts as a router.

That avoids building logic inside the model prompt.
